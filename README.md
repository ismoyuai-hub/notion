### 第二步：提交代码到 GitHub (三部曲)

修改并保存后，你需要通过终端（Terminal）或 Git Bash 执行以下三个核心命令。

假设你现在就在 `notion-widgets` 这个根目录下，请依次输入：

#### 1. 查看状态（可选，但推荐）

这会告诉你哪些文件被修改了（通常会显示为红色）。

Bash

```
git status
```

#### 2. 添加修改 (Add)

这个命令会把所有的修改放入“暂存区”（准备打包）。

Bash

```
git add .
```

*(注意：`add` 和 `.` 中间有一个空格，代表添加当前目录下所有变动)*

#### 3. 提交修改 (Commit)

这个命令是把暂存区的文件“打包”，并贴上一个说明标签。

Bash

```
git commit -m "优化了打卡组件的颜色和间距"
```

*(注意：引号里的内容是你对这次修改的备注，写给自己看的，中英文都可以)*

#### 4. 推送到远程 (Push)

这个命令是把本地的“包裹”真正上传到 GitHub 服务器。

Bash

```
git push
```

------

### 第三步：验证更新

当你看到终端显示 `Writing objects: 100% ...` 并且没有报错时，说明代码已经成功上传了。

**⚠️ 关键提示：关于缓存 (Cache)**

GitHub Pages 的部署通常需要 **1-3 分钟** 才能生效。而且，Notion 和浏览器都有很强的缓存机制。

如果你刷新 Notion 页面发现组件没有变化：

1. **等待**：先等 2 分钟。
2. **强制刷新**：在浏览器打开组件的直链（例如 `https://你的用户名.github.io/notion-widgets/reading-tracker/`），使用 `Ctrl + F5` (Windows) 或 `Cmd + Shift + R` (Mac) 强制刷新，确认网页版已经变了。
3. **Notion 端**：如果网页版变了但 Notion 没变，尝试在 Notion 组件块上点击右上角的 `...` -> `Reload`，或者重新粘贴一次链接。

### 💡 进阶小技巧：如何修改完想“反悔”？

如果你在本地改乱了，还没有执行 `git add`，想放弃所有修改，恢复到上一次提交的状态：

Bash

```
git checkout .
```

*(这会撤销所有未提交的本地修改，请谨慎使用！)*
